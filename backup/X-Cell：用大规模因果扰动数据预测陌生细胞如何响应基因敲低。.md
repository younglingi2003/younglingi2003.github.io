# 前言

标题：[X-Cell: Scaling Causal Perturbation Prediction Across Diverse Cellular Contexts via Diffusion Language Models](https://www.biorxiv.org/content/10.64898/2026.03.18.712807v1)

> 说明：截至 2026 年 7 月 20 日，我没有检索到该论文的 arXiv 版本，因此没有虚构 arXiv 链接，而是链接到 bioRxiv v1；[作者发布的完整 PDF 在这里](https://www.cdn.xaira.com/papers/X_CELL_V1_0316_final.pdf)。

标签：单细胞扰动预测；虚拟细胞；扩散语言模型；跨细胞类型泛化

摘要：这篇论文最本质的想法是：**不要只让模型学习“哪些基因经常一起表达”，而要用大规模基因敲低实验教它“敲低某个基因以后，细胞会怎样变化”，再把这种变化规律迁移到新细胞中。**作者为此构建了包含 2560 万个受扰动细胞的 X-Atlas/Pisces 数据集，并训练了 X-Cell：输入一组未扰动细胞和待敲低的基因，模型经过四轮类似“逐步填空”的扩散过程，生成扰动后的表达分布；其 49 亿参数版本 X-Cell-Ultra 在未见过的黑色素细胞祖细胞和原代 CD4⁺ T 细胞上取得了领先结果。不过，论文所谓“零样本”通常仍会使用目标域的未标注对照细胞进行测试时适配，而“LLM 级缩放规律”主要体现在训练损失上，不能简单理解为模型越大、真实生物学预测就会持续等比例提高。

阅读路线： **为什么需要因果扰动数据 → X-Atlas/Pisces 数据集 → X-Cell 如何生成扰动响应 → 多模态先验与损失函数 → 49 亿参数扩展 → 零样本究竟“零”在哪里 → 实验证据 → 最终评价。**

# Step 1：论文解读

## 1：它究竟想解决什么问题

给定一种细胞和一个基因，研究者希望提前知道：

> 如果把这个基因敲低，这个细胞的数千个基因会如何变化？

这对于药物靶点发现非常重要：如果一个疾病相关基因被抑制后，细胞表达状态向健康状态移动，它就可能是候选靶点；反之，如果某种干预激活了毒性或耐药通路，就应尽早放弃。

困难来自三个方面。

第一，同一个基因在不同细胞中作用可能完全不同。比如某个线粒体基因在癌细胞、iPSC 和激活 T 细胞中的下游响应可能差异很大。

第二，单细胞测序会破坏细胞，无法对同一个细胞同时测量“扰动前”和“扰动后”。模型面对的不是成对样本，而是两群细胞的分布：

$$
P(X_{\text{control}})\rightarrow P(X_{\text{perturbed}})
$$

第三，过去许多单细胞基础模型主要在普通表达图谱上预训练，学到的是“相关性”；而真正的 Perturb-seq 数据包含主动基因干预，能更直接地揭示“扰动→表达变化”的方向性关系。

但这里要谨慎理解“因果”：CRISPRi 实验确实提供了特定实验条件下的干预证据；模型把这种关系推广到新细胞时，仍然是在做统计迁移，并不等于它自动掌握了完整的分子因果机制。

## 2：数据底座 X-Atlas/Pisces

论文的重要性有一半来自模型，另一半其实来自数据。

X-Atlas/Pisces 包含 7 个全基因组 CRISPRi Perturb-seq 筛选：

- HCT116 结直肠癌细胞；
- HEK293T 肾来源细胞；
- HepG2 肝癌细胞；
- iPSC；
- 静息 Jurkat T 细胞；
- CD3/CD28 激活的 Jurkat T 细胞；
- 多谱系分化中的 iPSC。

最后一个筛选又包含 10 种分化细胞状态，因此合计覆盖 16 种细胞背景、约 2560 万个扰动细胞和超过 15.2 万个“扰动基因—细胞背景”组合。[论文 PDF](https://www.cdn.xaira.com/papers/X_CELL_V1_0316_final.pdf)

数据质量总体较高：7 个筛选的中位数约为每细胞 25,478 个 UMI、6,708 个被检测基因，中位目标基因敲低率为 78.7%。不同筛选中具有显著转录表型的扰动比例差别很大，从 HEK293T 的 25.32% 到 iPSC 的 72.80%。

一个很有价值的观察是： **在总细胞数相同的情况下，多测几种细胞背景，通常比在同一种细胞里继续堆细胞更有助于恢复已知生物网络。**这说明虚拟细胞的真正稀缺资源可能不是“测序细胞总数”，而是“不同背景下的干预覆盖度”。

此外，即使某些强扰动能跨细胞类型稳定出现，它们改变的具体下游基因重叠度依然不高。这进一步说明：预测扰动响应，本质上不是给每个基因记一个固定的效果向量，而是学习一个依赖细胞状态的函数。

## 3：X-Cell 的输入、输出和任务形式

一次训练样本包含：

- 一组 64 个未扰动对照细胞；
- 一组来自同一背景的扰动细胞；
- 被 CRISPRi 敲低的目标基因；
- 每个细胞的基因表达；
- 关于目标基因的外部知识。

模型词表最多包含约 19,408 个蛋白编码基因，但每个训练批次最多采样 4,000 个基因，以控制计算量。

输出不是“这个控制细胞唯一对应的扰动后细胞”，因为真实配对不存在；它输出一组预测细胞，再通过集合级损失，使整组预测细胞的分布接近真实扰动细胞分布。

这比单细胞逐一回归更符合实验事实。你可以把任务理解成：

> 已知一座城市现在的人口分布和一项新政策，预测政策实施后的整座城市，而不是硬说实施前的张三一定对应实施后的某个具体张三。

## 4：所谓“扩散语言模型”是怎样工作的

X-Cell 的扩散过程不是典型图像扩散中“从高斯噪声逐步生成图像”，而是一种 **掩码式、逐步填充的离散扩散**。

训练时，模型会随机选择 25%、50%、75% 或 100% 的基因位置，把这些位置的控制表达量替换成真实的扰动后表达量，并用二值掩码告诉模型哪些位置已经被“揭晓”。模型的任务是根据：

- 尚未改变的控制表达；
- 已揭晓的一部分扰动表达；
- 目标基因信息；

恢复完整的扰动表达谱。

推理时没有真实答案可供揭晓，因此过程改成：

1. 从全控制表达谱开始；
2. 预测完整的扰动表达谱；
3. 接受其中 25% 的预测值，填回输入；
4. 再预测一次，累计接受到 50%；
5. 继续到 75%；
6. 第四步生成最终结果。

论文显示，随着四轮推理推进，Pearson Δ 上升、MAE 下降，说明迭代自我修正确实有作用。

不过，这种设计存在一个值得注意的训练—推理差异：训练中间状态含有真实扰动值，而推理中间状态只有模型自己的预测。如果前几步预测有系统性错误，后续可能不断强化错误。论文展示了整体性能提升，但没有特别深入分析这种误差传播。

## 5：模型如何利用生物学先验

目标基因并不是一个孤立的编号。X-Cell 为每个扰动基因收集了多种先验表示：

- **GenePT**：从基因文字描述和文献中获得功能语义；
- **ESM-2**：从蛋白质序列中获得结构和生化信息；
- **STRING**：蛋白质相互作用网络；
- **DepMap**：不同癌细胞系对基因缺失的依赖性；
- **JUMP Cell Painting**：敲低基因后的细胞形态特征；
- **scGPT/基因嵌入**：从单细胞表达图谱中学习的基因关系。

这些表示通过交叉注意力注入 Transformer。直观上，模型在预测某个基因敲低效应时，可以同时“查阅”这个蛋白长什么样、与谁相互作用、在哪些癌细胞中必需、会造成什么形态变化，以及文献如何描述它。

随机打乱单项先验的实验显示，DepMap 和 STRING 对性能影响最大；注意力权重则较多落在 STRING 和 ESM-2 上。说明结构化功能网络比纯文字描述更直接地帮助扰动预测。

但要避免把注意力权重当成机制解释。注意力能告诉我们模型使用了哪类信息，却不能证明某条通路就是预测成立的生物学原因。

## 6：七类损失与“保守塌缩”

扰动预测有一个很常见的作弊方式：预测“几乎没有变化”。

因为绝大多数基因在一次扰动后变化都很小，所以输出接近控制细胞往往能获得很低的平均误差，却完全错过真正重要的差异表达基因。作者称其为“保守塌缩”。

X-Cell-Ultra 的完整损失设计包含七个部分：

1. MMD/能量距离：匹配整组预测细胞和真实细胞的联合分布；
2. fold-change CCC：同时约束变化方向和幅度；
3. per-gene CCC：保持单基因在细胞群中的异质性；
4. Δ-norm：防止预测变化幅度被压小；
5. pseudobulk Δ-MSE：约束平均表达变化；
6. 对比损失：让相同扰动的细胞表示靠近；
7. 先验重建损失：防止交叉注意力忽略外部知识。

实际训练阶段并不是始终同时使用全部七项：55M 版本和 Ultra 的不同训练阶段使用了不同组合。Ultra 后期还移除了 per-gene CCC。

消融实验很有说服力：在 17.5 亿参数模型上，单纯使用 MMD、Pearson 和 cosine 时，预测变化幅度只有真实值的约 22%，DE Pearson 仅 0.050；加入 CCC、移除隐藏残差捷径并使用共享基因嵌入解码后，幅度比例提高到 51%，DE Pearson 提高到 0.327。

换句话说，论文真正有技术含量的地方不只是“用了扩散”，还在于它认真处理了扰动预测中特别隐蔽的“看起来误差很低、其实什么也没预测出来”的问题。

## 7：从 5500 万到 49 亿参数

基础版 X-Cell 有 12 层、512 维隐藏空间和约 5500 万参数，并从 scGPT 权重初始化。

X-Cell-Ultra 有 34 层、2560 维隐藏空间和 48.7 亿参数。为了让这种规模稳定训练，作者采用了大模型中常见的设计：

- Pre-LN 与 RMSNorm；
- SwiGLU；
- QK-Norm；
- 输出解码器与基因嵌入权重共享；
- 分布式参数分片与激活检查点。

Ultra 不再依赖 scGPT 的观察性预训练权重，而是从大规模干预数据直接训练。训练分两阶段：

1. 先在每种细胞中表型最强的前 5% 扰动上训练；
2. 再用完整数据集低学习率训练。

最终训练语料约为 2028 万细胞和 151,180 个扰动—背景组合；运行使用 64–128 张 H200 GPU，报告的模型 FLOPs 利用率约 41%。

## 8：“零样本”到底零在哪里

论文中有三种不同含义的零样本，需要分开看。

**Jurkat 激活状态：** 模型只用静息 Jurkat 的扰动数据微调，然后预测激活 Jurkat 中的扰动效应。这里没有使用激活态扰动标签，属于较标准的跨状态零样本。

**黑色素细胞祖细胞和原代 T 细胞：** 这些细胞的扰动数据没有进入训练，但模型会使用目标细胞中的未标注对照细胞进行 Test-Time Adaptation。它先用约 200–1024 组目标域对照细胞进行 control-to-control 自监督适配，再恢复交叉注意力预测扰动。

因此更准确的叫法是：

> 零扰动标签的无监督目标域适配，而不是完全不接触目标细胞数据的严格零样本。

 **药物预测：** 模型把单靶点抑制剂的已知作用靶点当成“被敲低基因”，进而预测药物表达响应。这只在 12 种中等剂量、单靶点抑制剂上测试，尚不能说明它能处理多靶点药物、剂量效应、代谢产物或非抑制性机制。

# Step 2：实验部分

## 1：实验设置和对照方法

论文包含三类主要评估：

- **同背景留出扰动：** iPSC-200、HepG2-200；
- **外部数据微调：** Replogle–Nadig、Parse-1M；
- **跨背景零扰动标签：** Tahoe 药物子集、激活 Jurkat、黑色素细胞祖细胞、原代 CD4⁺ T 细胞。

基线包括 Cell2Sentence、STATE、scGPT，以及两个很重要的简单基线：

- Control Mean：假设扰动没有效果；
- Perturbation Mean：把该扰动在其他细胞背景中的平均变化直接搬到目标细胞。

公平性并不完全一致。iPSC/HepG2 实验中：

- X-Cell 使用约 1000 万个预训练细胞；
- STATE 只在约 550 万细胞的子集上训练；
- Cell2Sentence 因计算成本过高，只报告零样本结果。

Replogle–Nadig 和 Parse-1M 中，各模型使用相同划分进行微调，比较相对更公平。

## 2：数据集自身是否包含可靠生物学

把 1 到 7 个筛选逐步合并后，模型对 CORUM 蛋白复合物和 STRING 相互作用的召回率单调上升。同样数量的细胞分散在多种背景中，比集中在单一背景中更有效。

聚类结果也符合生物学：

- 跨细胞稳定的强效扰动主要富集在线粒体呼吸、氧化磷酸化等基础过程；
- HepG2 特异扰动富集于缺氧和葡萄糖代谢；
- iPSC 特异扰动富集于神经嵴分化、发育谱系和上皮—间质转化。

这部分证明 X-Atlas/Pisces 不只是“大”，还确实捕捉了有意义的背景依赖调控。

但值得注意的是，即使强扰动的跨背景可检测性很高，其下游特征的绝对 Jaccard 重叠依然较低——这恰好说明跨细胞预测非常困难，也说明简单迁移平均扰动向量通常不够。

## 3：标准基准结果

下表列出最直观的 Pearson Δ 和 MAE。Pearson Δ 衡量预测与真实扰动 fold-change 的相关性；MAE 越低越好。[完整结果见论文 Table B7](https://www.cdn.xaira.com/papers/X_CELL_V1_0316_final.pdf)。

| 数据集 | 最强基线 Pearson Δ | X-Cell Pearson Δ | 最强基线 MAE | X-Cell MAE |
|---|---:|---:|---:|---:|
| iPSC-200 | STATE 0.1042 | **0.5104** | scGPT 0.0304 | **0.0257** |
| HepG2-200 | STATE 0.1014 | **0.4756** | scGPT 0.0299 | **0.0256** |
| Replogle–Nadig | STATE 0.4563 | **0.5137** | scGPT 0.0804 | **0.0682** |
| Parse-1M | STATE 0.5360 | **0.5858** | scGPT 0.0941 | **0.0850** |

“五倍提升”主要来自内部 iPSC/HepG2 留出集：0.51 对 0.10、0.48 对 0.10，确实很显著。

但外部数据上的提升更温和：Replogle–Nadig 是 0.514 对 0.456，Parse-1M 是 0.586 对 0.536。这仍是进步，只是不应把内部集上的“五倍”推广成普遍结论。

另外，X-Cell 并非每项指标都赢。例如 Parse-1M 的 Overlap@N 是 STATE 的 0.3044 对 X-Cell 的 0.2500；DE Direction Match 也是 STATE 略高。说明不同指标反映不同目标：X-Cell 更擅长恢复 fold-change 幅度、排序和整体表达，而不总是找回最多的 top-DE 基因。

## 4：药物迁移和 T 细胞失活预测

在 Tahoe-100M 的 50 个癌细胞系、12 种单靶点抑制剂上，X-Cell 没见过这些药物扰动，只根据药物靶点基因进行预测：

- Pearson Δ：0.31，对比 STATE 的 0.22；
- MAE：0.08，对比 STATE 的 0.16。

这是一个不错的“遗传扰动→药物扰动”概念验证，但任务被严格限制在单靶点抑制剂和中等剂量，因此离真实药理预测仍有很大距离。

在激活 Jurkat 中，作者构造了“失活指数”：扰动后的激活细胞越接近静息细胞，指数越高。CD3D、CD3E、CD3G、CD247、ZAP70 等经典 TCR 调控因子得到高分，符合预期。

X-Cell 还提名了 LRBA、APPL2、WDR53 等潜在调控因子，并比“直接搬运静息态扰动向量”的线性基线更准确。APPL2 和 LRBA 随后在独立原代 T 细胞数据中获得支持。

不过这属于 **独立数据复核** ，不是作者新做的前瞻性湿实验验证；论文尚未证明干预 APPL2 或 LRBA 能产生预期的治疗效应。

## 5：X-Cell-Ultra 的未见细胞实验

黑色素细胞祖细胞被完全排除在预训练之外，测试包含 1,341 个扰动。经过只使用对照细胞的 TTA 后：

- X-Cell-Ultra 的 Pearson Δ 约为 0.29；
- STATE 约为 0.13；
- 55M X-Cell 约为 0.08；
- Ultra 在 DE 方向和 MAE 上也最好。

这说明增大数据、模型和改进训练方案后，模型确实可以越过一个新的细胞谱系。

原代 CD4⁺ T 细胞实验包括 2 名供体、静息/刺激 8 小时/刺激 48 小时三个条件和 291 个 T 细胞调控基因。Ultra 在全部供体—条件组合上优于 STATE 和小模型，特别是在激活状态下。

但其绝对 Pearson Δ 大多仍在约 0.1 的量级。也就是说，它比现有方法更好，却远没有达到“可以高精度替代真实实验”的程度。论文证明的是有用的排序和初筛能力，不是完整的数字细胞孪生。

## 6：缩放规律、可复现性与实验结论

缩放实验在相同 Replogle–Nadig 数据上训练了 83M、216M、543M、1.6B 和 3.1B 五个模型：

| 参数量 | 验证损失 | DE Pearson |
|---:|---:|---:|
| 83M | 5.625 | 0.291 |
| 216M | 5.376 | 0.304 |
| 543M | 5.203 | 0.313 |
| 1.6B | 5.057 | **0.315** |
| 3.1B | **5.003** | 0.312 |

训练损失的缩放指数为 0.322，确实落在语言模型文献的范围内；但测试损失指数只有 0.033，DE Pearson 在 16 亿参数后基本饱和，甚至从 0.315 略降到 0.312。

所以最诚实的解读是：

> 更大的模型能更好地拟合扰动数据，但真正的生物学泛化已经受到“独立扰动—背景组合不够多”的限制。

这其实比“生物学也遵守大模型 scaling law”更重要。对于单细胞模型，把每个基因值都数成一个 token 会产生巨大的 token 数，但这些 token 并非独立知识；真正有效的数据单位可能是一个新的干预、细胞类型、供体或环境条件。

此外，截至目前，官方 Hugging Face 模型卡和 GitHub 仓库仍明确标注“模型权重与推理代码即将发布”，因此尚不能独立复现其核心 49 亿参数结果。[Hugging Face 模型卡](https://huggingface.co/Xaira-Therapeutics/X-Cell)、[GitHub 仓库](https://github.com/xaira-therapeutics/x-cell)

# Step 3：other

## 1：重点

这篇论文最重要的贡献不是单独提出了一个 49 亿参数模型，而是把三个要素连成了一条完整路线： **大规模干预数据、背景依赖的分布生成、多模态生物学先验。** 其中最值得记住的结论是：虚拟细胞的瓶颈很可能不是总细胞数，也不只是模型参数量，而是覆盖了多少真正不同的“扰动×细胞背景”组合。X-Atlas/Pisces 显示，多增加一种细胞背景通常比继续深挖同一种细胞更有价值；缩放实验也显示，模型超过 16 亿参数后，生物学指标已经明显受数据多样性限制。

技术上，X-Cell 对“保守塌缩”的处理也很有价值：不能只用 MAE 判断扰动模型，因为一个几乎不预测变化的模型也可能得到漂亮的 MAE；必须同时衡量差异表达方向、幅度、基因排序、扰动可区分性和整个人群分布。

## 2：意见

作为严格审稿人，我最关键的意见是： **论文没有通过严格的受控实验，把 X-Cell-Ultra 的提升归因于“模型规模”。** Ultra 相比基础版同时改变了参数量、网络结构、训练数据量、细胞背景数量、损失函数、反塌缩设计、课程学习和测试时适配；而真正的缩放实验最高只到 31 亿参数，并且 DE Pearson 在 16 亿参数后已经饱和。因此，现有证据能够支持“整套 Ultra 系统更强”，却不能支持“扩大到 49 亿参数本身带来了跨细胞泛化”。需要在相同数据、相同目标、相同 TTA 和相近计算预算下比较不同参数量，并报告无 TTA 结果，才能坐实论文标题中最强的 scaling 与 zero-shot 主张。

## 3：相关工作

1. [Predicting cellular responses to perturbation across diverse contexts with STATE](https://nips.cc/virtual/2025/129565)（NeurIPS 2025）：STATE 同样把细胞当作集合，通过分布匹配预测扰动后细胞群，并以超过一亿个扰动细胞训练。它是 X-Cell 最直接的前身和主要基线；X-Cell 在此基础上加入了掩码扩散、多模态先验和专门的反塌缩目标。

2. [PerturbDiff: Functional Diffusion for Single-Cell Perturbation Modeling](https://arxiv.org/abs/2602.19685)（2026）：PerturbDiff 也认为控制与扰动细胞无法配对，因此直接对“概率分布”进行扩散，而不是生成某个固定的细胞响应。它进一步试图描述同一已知条件下由微环境、批次等隐藏变量造成的多个可能响应分布，理论味道比 X-Cell 更强。

3. [OCOO-T: A Simple and Scalable Virtual Cell Model for Transcriptional Perturbation Response Prediction](https://www.biorxiv.org/content/10.64898/2026.06.08.731000v1)（bioRxiv，2026 年 6 月）：这是 X-Cell 之后出现的一个很有意思的反方向工作。它不堆叠复杂的先验编码器，而是用朴素 Transformer 和连续时间 rectified flow 直接生成表达谱，在 Tahoe-100M、Replogle 和 PBMC 上取得竞争性结果。它提出了一个值得认真对待的问题：X-Cell 的优势究竟来自复杂的多模态先验，还是主要来自数据规模、去噪目标和训练工程？
